# Gate Hardening Checklist

This checklist audits the maturity of each activated Progressive Gate Schedule
row in [README.md](README.md). It is intentionally gate-focused: OpenClarion's
product direction remains intelligent alert analysis, while this document
keeps the CI and governance framework honest as the implementation broadens.

The companion checker, `make gate-hardening-check`, proves that every activated
Schedule row has exactly one record here and that records use concrete evidence
instead of placeholders.

## Maturity Levels

| Level | Meaning |
|-------|---------|
| baseline | The gate blocks its main drift mode but still has a narrow rule surface. |
| hardened | The gate has committed positive and negative coverage and is wired into local and remote validation. |
| mature | The gate covers the current production contract with focused tests, drift checks, and documented operating limits. |
| manual | The gate is a retained evidence or live-smoke workflow that cannot safely run in normal CI. |
| partial | The gate records useful evidence but cannot accept the architectural baseline yet. |
| replaced | The gate's original implementation was retired after equivalent or stronger coverage landed elsewhere. |
