# Documentation

This directory contains the project documentation authority for OpenClarion.

## Reading Order

1. [README](../README.md) - project overview and scope
2. [ADR index](adr/README.md) - architecture decisions
3. [Design overview](design/README.md) - system architecture
4. [Roadmap](roadmap/tasks.md) - implementation sequencing
5. [Development workflow](../DEVELOPMENT_WORKFLOW.md) - local development and review
6. [Contributing](../CONTRIBUTING.md) - contribution policy
7. [Security](../SECURITY.md) - vulnerability and security baseline

## Documentation Authority

| Layer | Authority |
|-------|-----------|
| Project position | root `README.md` |
| Architecture decisions | `docs/adr/` |
| Product flow | `docs/design/interaction-flows/master-flow.md` |
| Implementation design | `docs/design/` |
| Work sequencing | `docs/roadmap/tasks.md` |
| Contribution and governance | root governance files |

When documents conflict, the order is:

1. accepted ADRs
2. root project governance files
3. design specifications
4. roadmap and phase plans
5. historical notes
